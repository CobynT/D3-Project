NBA Performance by Age – Project Documentation

Interactive D3 visualization exploring how NBA scoring, rebounding, and playmaking change as players age.

Part 1 – Motivation & Questions

1. Motivation and Dataset

I chose an NBA dataset because I’ve always been fascinated with the combination of sports and statistics. One of the most common debates in basketball is: “When do players actually peak, and when does their game start to decline?”

To explore this question, I used a Kaggle dataset containing historical NBA player data. The nba_players.csv file includes attributes such as:

  • age – player age
  • gp – games played
  • pts – points per game
  • reb – rebounds per game
  • ast – assists per game
  • team_abbreviation, season, etc.

2. Initial Questions

My main big-picture question was:

• At what age do NBA players reach their peak performance, and when do they start to decline?

As I explored the data, related questions came up:

  • How does the peak age differ for specific stats? For example, do points, rebounds, and assists peak at the same age?
  • When a player scores less as they get older, do other stats (like assists) increase because they may pass more instead of scoring?
  • Is there a clear “peak age” for each stat, or is the decline more gradual?

Part 2 – Design Process

3. Design Process & Iterations

3.1 First Prototype

I started with a very simple design: a single bar chart showing average points per game by age. The chart had:

  • One bar per age
  • Plain blue color for all bars
  • No interactivity or filters

[Screenshot idea: first simple bar chart of PTS vs age]

3.2 Major Changes and Improvements

As I interacted with the first chart, I realized it answered only part of my question. From there I made a few key changes:

  • Metric dropdown: I added a dropdown that lets the user switch between Points (PTS), Rebounds (REB), and Assists (AST). This allows comparisons of how different parts of a player’s game change with age.
  • Position filter: I then added a position filter so you can choose between positions like guards, forwards, centers (or all positions). This helps explore whether, for example, big men peak differently than guards.
  • Minimum games played filter: I added a slider that lets users select a minimum number of games played. This was important so that rare outliers with only a few games don’t distort the averages.
  • Color improvements: I updated the color palette so each stat uses a distinct accent color that “pops” more and feels more professional, instead of all stats sharing a plain blue.

Part 3 – Design Decisions

4. Rationale for Design Choices

4.1 Why a Bar Chart?

Comparing ages directly was the most important task, so I chose a bar chart. With one bar per age, it is very easy to:

  • See which age has the highest average value
  • Compare neighboring ages and spot where decline begins
  • Notice general “shape” trends (rising, peak, falling)

4.2 Interaction Design

  • Stat dropdown: Lets users compare how different aspects of performance (scoring, rebounding, playmaking) change with age beyond just points.
  • Position filter: Supports questions like “Do guards have a different peak age than centers?” without overwhelming the main view.
  • Minimum games slider: Ensures the averages represent real playing time instead of being skewed by players who only appeared in a few games.

4.3 Color & Aesthetics

I adjusted the colors to make the visualization cleaner and more readable:

  • Dark background to keep focus on the data and make the bars stand out.
  • Each stat uses its own accent color, which helps the user immediately recognize whether they are looking at points, rebounds, or assists.
  • Subtle hover effects and labels to keep the design professional but still clear.

Part 4 – Findings

5. Insights & Patterns

5.1 Points vs Age

When looking at points per game by age, the chart suggests that players tend to peak around ages 27–28. After that:

  • There is a small decline right after the peak.
  • A larger drop appears when crossing from about age 30 to 31, showing the more serious part of the decline.

5.2 Rebounds vs Age

Rebounds show a similar pattern to points, but the decline seems a bit less severe. The performance drops with age, but not quite as sharply as scoring.

5.3 Assists vs Age

Assists behave differently:

  • There is not a clear sharp “peak” like with points — assists stay more steady through the late 20s and early 30s.
  • The decline becomes more visible only once players reach their mid-30s.

This supports the idea that as players get older, their raw scoring might decline, but their playmaking and passing can stay strong for longer.

5.4 Effect of Minimum Games Filter

When increasing the minimum games played, the bars sometimes shift and can make some older ages look slightly better. This suggests that:

  • Very small sample sizes (players with only a few games) can distort the averages.
  • Filtering out low-game seasons may highlight more “real” rotation players and smooth out noisy outliers.

Part 5 – Usage & Audience

6. How to Use the Visualization

The main goal of the visualization is to let users explore how NBA performance changes with age, and to make it easy to:

  • See the peak age for points, rebounds, and assists.
  • Compare how quickly different stats decline with age.
  • Filter by position and games played to reduce noise.

The intended audience is more on the casual side: fans, students, and anyone curious about NBA careers and aging, rather than only professional analysts. The interface is kept simple with just a few controls so it’s easy to understand without reading a manual.
